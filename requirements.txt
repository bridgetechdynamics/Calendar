You are a world class user interface designer and time/date expert. We need a dynamic calendar for our website implemented in bare bones, vanilla Javascript that will allow us to:
1) display the current month
2) change the either the month and/or year independently via
    a) month dropdown
    b) Prev/Next buttons for changing month
    c) year dropdown
    d) Prev/Next buttons for changing year
    e) Today button
3) insert events per day at calendar instantiation via JSON
4) allow browser to add ad hoc events with intuitive and direct controls via JSON (reject duplicate id)
5) allow browser to remove ad hoc events by unique id with intuitive and direct controls via JSON
6) the current day (Today) should always be highlighted
7) the selected day will be seperately highlighted

Additionally, the calendar needs to:
1) account for leap years
2) recognize the most common USA holidays and allow their respective date cells to be optionally labeled
3) be implemented in Javascript without additional need for external libraries
4) be generated in one complete page named calendar.html (we will slice up and style CSS in post)
5) have a modern look and feel
6) have an eye catching header that displays given month and year

The JSON structure will contain:
1) id: required unique integer
2) label: required non-unique string
3) href: optional non-unique hypertext reference
4) date: required non-unique date ("YYYY-MM-DD")

Here is an example JSON datastructure that I would use to instantiate the calendar with:
[
    {"id":1,"label":"Event 1","href":"https://…","date":"2024-03-25"},
    {"id":2,"label":"Event 2","date":"2024-03-25"},
    {"id":3,"label":"Event 3","href":"https://…","date":"2024-03-26"}
]

I should be able to add more events via a Javascript function after the page is loaded with a similar JSON datastructure:
[
    {"id":7,"label":"Event 4","href":"https://…","date":"2024-03-27"},
    {"id":8,"label":"Event 5","date":"2024-03-28"},
]

Add I should be able to delete events via a Javascript function after the page is loaded with a JSON datastructure like so:
[
    {"id":2},
    {"id":7},
]

Final considerations:
1) we can probably only fit 3 events per cell so offer a "more" control if we go over that threshold
2) we a want a pure calendar grid with event badges in each date cell 
3) we also want the ability for a list/details panel modal to appear when clicking a date
    a) will display all events for that day in a display panel
    b) when an event is clicked in the panel, an edit panel will be swapped in
        i) the edit panel will use form elements to allow the event attributes to be modified
        ii) the edit panel will need a control to save the changes and one to cancel (both will swap back to an updated display panel)
    c) each event can optional have an 'x' like control that deletes the event by swapping in a confirmation panel
        i) the confirmation panel will provide controls to delete (remove the event) or cancel (take no action)
        ii) both actions will swap back to the updated display panel
    b) a close panel button will be required to dismiss the panel
4) we would like for ad hoc events to survive reloads
5) we would like a simple public API surface (addEvents, removeEvents, setMonthYear, getEvents, etc.) and a consistent return value or promise
